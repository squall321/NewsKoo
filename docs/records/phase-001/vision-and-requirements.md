# Phase 001 — Vision & Requirements Brief

## Executive Summary
NewsKoo는 전 세계 업계 유머 콘텐츠를 **저비용 번역·재구성 파이프라인**을 통해 한국 독자에게 제공하는 것을 목표로 한다. 본 문서는 페이즈 001 활동을 통해 도출한 제품 비전, 핵심 가치, 요구사항을 정리하여 이후 299개 페이즈의 방향성을 고정한다.

## Guiding Principles
1. **초저비용 운영**: RTX 5070 Ti (16GB VRAM) 기반 로컬 추론과 오픈소스 모델을 우선하며, 상용 API는 비용·약관 검토 후 보조적으로만 사용한다.
2. **원문 재창작**: Reddit 등 소스의 핵심만 추출하여 NewsKoo 톤으로 재작성해 약관 위반을 피하고, 문화적 적합성을 높인다.
3. **검수 친화성**: 자동 번역/요약 결과를 사람이 빠르게 편집·승인할 수 있는 검수 도구 UX를 최우선으로 설계한다.
4. **모바일·데스크톱 동등 경험**: 반응형 UI와 향후 React Native 기반 배포를 염두에 둔 컴포넌트 전략을 채택한다.

## Target Personas & Needs
| 페르소나 | 주요 니즈 | 성공 기준 |
| --- | --- | --- |
| 큐레이터(운영자) | 자동 수집본을 빠르게 검수/편집/게시 | 하루 30건 이상 승인 가능, 5분 내 게시 |
| 일반 독자 | 장르별 최신 유머, 모바일 친화 UI | 체류시간 3분+, 재방문 주 2회 |
| 모바일 앱 사용자 | 앱에서도 동일 콘텐츠/광고 경험 | 푸시 알림 CTR 10% 이상 |

## Functional Requirements (FR)
1. **FR-001: 크롤링 후보 풀** — Reddit/9GAG/트위터 리스트에서 컨텐츠 핵심을 수집, 메타데이터(업계, 주제, 출처 URL)를 포함한다.
2. **FR-002: 로컬 번역/재구성 파이프라인** — Hugging Face LLM + 텍스트 가이드로 한국어 초안을 생성한다.
3. **FR-003: 검수 대시보드** — 자동 초안을 큐레이터가 비교·편집·게시할 수 있는 UI.
4. **FR-004: 콘텐츠 퍼블리싱 API/페이지** — 초기에 서버 렌더링 없이 정적 페이지에 게시하고, 이후 API 추가 시 역호환한다.
5. **FR-005: 광고/수익 추적 준비** — 페이지 구조에 광고 슬롯 placeholder를 정의한다.

## Non-functional Requirements (NFR)
- **NFR-001 비용 제한**: 월 10만원 내외의 인프라/전력 비용을 목표로, GPU 1대와 소규모 VPS만 사용.
- **NFR-002 번역 품질**: 인간 검수 전 BLEU 혹은 주관적 점수 70/100 이상.
- **NFR-003 응답성**: 주요 페이지 LCP 2.5초 이하 (모바일 4G 가정).
- **NFR-004 보안/법무**: 소스 재작성 로그 보관, 출처 명시 메타데이터 관리.

## Translation & Rewriting Constraints
- `docs/strategy/translation-guardrails.md`의 지침을 전 페이즈에서 준수한다.
- 로컬 추론 모델 후보: Mistral-7B, LLaMA-3-8B, BLOOMZ-7B; 필요 시 QLoRA/4bit 양자화.
- 콘텐츠 재작성 템플릿: “핵심 상황 요약 → 업계 맥락 → 한국 독자 맞춤 punchline” 구조를 권장한다.

## Dependencies & Open Questions
- 번역 모델 파인튜닝 필요성 평가 (Phase 014 PoC 결과에 의존).
- 광고 게재 파트너(AdSense, 애드핏 등) 정책 재확인.
- Reddit 약관 상 데이터 사용 범위와 재작성 정도의 허용치.
