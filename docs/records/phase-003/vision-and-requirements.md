# Phase 003 — Market Research Synthesis

## Executive Summary
정성(인터뷰 15건)·정량(설문 512명) 데이터를 결합해 **운영자, 일반 독자, 광고 파트너** 세 축의 니즈를 재정의했다. 시장 조사 결과, 번역 정확도보다 "출처 투명성"과 "전문 맥락"이 재방문율과 광고 전환율에 더 큰 영향을 미치며, 이는 Phase 011 UI/UX 설계와 Phase 018 관측 파이프라인에 즉시 반영되었다.

## Insights & Priorities
1. **출처 투명성**이 뉴스 신뢰도의 35%를 설명 → 검수 UI에 추론 로그를 기본 노출.
2. 모바일 사용자는 "짧은 요약 + 원문 링크"를 가장 신뢰 → CTA 구성에 원문 버튼 필수.
3. 큐레이터는 세션당 8건 이상 처리하려면 번역 재실행 시간을 제어해야 함 → 로컬 모델 캐시 우선순위 상향.
4. 광고주는 카테고리보다 "업계/직무" 세그먼트를 원함 → KPI도 직무 기준으로 재정의.
5. 로컬 AI 제약(16GB VRAM)으로 LLaMA-3-8B 4bit 구성이 안정적 → 모델 교체 시 검증 체크리스트 필요.
6. Reddit 약관 대응을 위해 원문 텍스트 보관 기간을 30일로 축소 → 감사 로그만 장기 저장.

## Requirements Update
### Functional
1. **FR-011: 출처 신뢰 패널** — 검수 화면에 출처/번역 로그/모델 버전을 즉시 노출.
2. **FR-012: 직무 기반 세그먼트 토글** — 독자 Facing 페이지에서 직무별 카드 필터 제공.
3. **FR-013: 원문 CTA 매크로** — 재작성 카드 하단에 원문 링크와 재작성 이유를 자동 표기.
4. **FR-014: 모델 캐시 제어** — 큐레이터가 번역 재실행 기준(온디맨드/자동)을 선택.
5. **FR-015: 투명성 샘플 Export** — Transparency API 미구현 상태에서도 CSV/PDF 로그를 추출.

### Non-functional
- **NFR-008 신뢰 지표**: 출처 표시 누락률 0.5% 이하, 로그 생성 24시간 내 공개.
- **NFR-009 로컬 성능**: LLaMA-3-8B 4bit 기준 추론 지연 6초 이하.
- **NFR-010 데이터 보존**: 원문 텍스트 30일, 재작성 로그 365일 보관.

## Dependencies
- FR-011~FR-014는 Phase 011 시스템 디자인과 Phase 012 API 설계 착수 시 스펙 반영.
- FR-015, NFR-008은 Phase 018 관측/로그 파이프라인과 Phase 024 파트너 온보딩과 직접 연결.
- 표본 확대 Action Item은 Phase 005(Research Ops)로 이관해 직무별 가중치를 정교화한다.
