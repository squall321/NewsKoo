# Phase 002 — Strategy, Vision & Persona Brief

## Executive Summary
Phase 002는 Phase 001에서 정의한 제품 비전을 기반으로 **페르소나 모델링 정교화**와 전략 가설을 업데이트하였다. 정성(심층 인터뷰 12건)과 정량(커뮤니티 설문 486명) 데이터를 결합하여 행동 패턴/기술 제약/수익 기대를 재정의하고, 워크숍에서 합의된 로드맵 우선순위를 반영해 요구사항과 의존성을 조정했다.

## Research Synthesis
- **정성 리서치**: 큐레이터, 콘텐츠 소비자, 광고 파트너 후보 총 12명을 대상으로 워크플로 관찰과 과업 기반 인터뷰를 실시해 수집/검수/게시 단계의 마찰 지점을 정리.
- **정량 리서치**: 커뮤니티 뉴스레터 구독자 486명 대상 설문으로 콘텐츠 길이, 알림 빈도, 과금 의향을 측정(응답률 38%).
- **주요 인사이트**:
  1. 게시 속도보다 **정확한 출처 표기**와 "번역 느낌 최소화"가 재방문 지표에 더 크게 기여.
  2. 큐레이터는 30분 미만의 집중 세션에서 6~8건 검수 가능 → UI 내 배치/프리뷰 효율성 강화 필요.
  3. 모바일 사용자의 62%가 "짧은 카드형"을 선호, 데스크톱 사용자의 54%는 "스트림 + 상세"를 선호하여 반응형 템플릿 분기 필요.
  4. 광고 파트너는 저비용 로컬 AI 활용 로그(모델명, 추론일자) 공유 시 브랜드 세이프티 우려 감소.

## Target Personas (5+)
| 페르소나 | 니즈/동기 | 장애요인 | 성공 지표 |
| --- | --- | --- | --- |
| **OP-큐레이터** (사내 운영) | 큐레이션 후보를 빠르게 검수/편집/게시 | 비교/프리뷰 탭 전환, 번역 오류 | 세션당 승인 8건, 10분 이내 게시 |
| **콘텐츠 에디터** (Freelance) | Tone/Style 가이드를 따라 2차 편집 | 원문 맥락 부족, 저작권 우려 | 편집당 수정 횟수 < 2회, QA 통과 95% |
| **밈 애호가 독자** | 출근길 5분 내 웃긴 업계 밈 소비 | 광고 과다, 모바일 스크롤 피로 | 세션 길이 3분+, 저장/공유 비율 12% |
| **주니어 PM 독자** | 해외 트렌드를 한국어로 빠르게 이해 | 번역체, 컨텍스트 부족 | CTR 8%+, 주당 재방문 2회 |
| **광고·스폰서 파트너** | 브랜드 세이프티, 투명한 소싱 기록 | 법무 리스크, 파이프라인 가시성 부족 | 검수 로그 공유, 월 리텐션 90% |

## Updated Requirements
### Functional Updates
1. **FR-006: 출처 추적 메타데이터 패널** — 큐레이터가 출처/번역 로그를 즉시 검증하고 편집본에 자동 삽입.
2. **FR-007: 카드형/스트림형 레이아웃 스위처** — 화면 폭과 사용자 설정에 따라 2종 레이아웃을 제공.
3. **FR-008: 워크플로 타이머 및 배치 추천** — 큐레이터 세션당 처리량을 모니터링하고 우선순위 큐를 추천.
4. **FR-009: 파트너 뷰어** — 광고 파트너가 모델 사용 로그와 게시물 샘플을 확인하는 공유 페이지.
5. **FR-010: 인사이트 태깅** — 정량 설문 결과를 콘텐츠 카드 메타데이터(톤, 길이, CTA)로 저장하여 AB 테스트 가능.

### Non-functional Updates
- **NFR-005 접근성**: 카드형 뷰 기준 키보드 탭 이동 거리 12포커스 이하.
- **NFR-006 투명성 로그**: 각 게시물에 사용된 모델, 추론 시간, 검수자 ID를 24시간 내 대시보드에 반영.
- **NFR-007 워크숍 합의 추적**: 페이즈별 합의율 90% 이상 유지, 이탈자 피드백을 의사결정 로그에 기록.

## Dependencies & Backlog Notes
- FR-006~FR-010은 Phase 011(Frontend 시스템 디자인)과 Phase 012(Backend API)에서 설계/개발 병렬 진행 필요.
- NFR-006 구현을 위해 Phase 018(Observability)에서 로그 파이프라인 우선순위 상향.
- 파트너 뷰어(Fr-009)는 Phase 024(파트너 온보딩)과 데이터 공유 정책 정렬 필요.
- 정량 리서치 추가 표본(디자이너 커뮤니티 200명)은 Phase 005 Research Ops 지원 필요.
- Transparency API 프로토타입(`/transparency/logs`)을 백엔드에 배치해 파트너와 공유 가능한 로그 포맷을 검증한다.

## Workshop Outcomes
- Product/Content/Engineering/Legal/Data 11명 참석, 10명 찬성(91%).
- 반대 1건: "카드형/스트림형" 동시 출시 범위 축소 제안 → Phase 011에서 MVP 범위 검토 Action Item 발행.
- 모든 페르소나 정의와 요구사항은 `translation-guardrails.md` 준수 및 저비용 로컬 AI 가정 하에 평가되었다.
