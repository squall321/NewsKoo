# Phase 015 - 전략 및 비전 정립

## Objective
핵심 사용자 세그먼트가 체감할 수 있는 가치 제안을 언어·비주얼·서비스 레벨로 구체화해 “왜 NewsKoo인가?”라는 질문에 일관된 답을 만든다. 경쟁사 대비 차별 가치, 사용자 페인 포인트, 프리미엄 지불 의향을 정교하게 정의해 다음 페이즈의 메시징·수익 모델 설계에 입력한다.

## Tasks
1. **세그먼트별 페인 포인트·동기 재검증**
   - Phase 013~014 리서치 산출물에서 상위 페인 포인트를 추려 사용자 인터뷰·설문으로 우선순위를 재평가한다.
   - JTBD(할 일) 프레임워크와 Value Proposition Canvas를 활용해 기능/감성/사회적 가치를 구분한다.
2. **차별 가치 패키지 설계**
   - 콘텐츠 큐레이션, 로컬라이징 품질, 커뮤니티 신뢰 등 3~4개의 핵심 가치 테마를 정의하고, 각 테마별 증거(Feature, Proof, Benefit)를 작성한다.
   - 텍스트·비주얼 레퍼런스를 포함한 Messaging Pillar 문서를 작성해 브랜드·마케팅 팀이 재사용하도록 한다.
3. **실험 가설 및 MVP 정의**
   - 가치 제안을 검증할 A/B 테스트 또는 Concierge MVP 가설을 최소 2건 정의하고, 필요한 데이터/도구/예산을 명시한다.
   - 가설별 성공 신호, 실패 시 피봇 경로, 의존 페이즈(016~020)를 표로 작성한다.
4. **워크숍/리뷰 라운드**
   - C-level 및 콘텐츠 리드가 참석하는 리뷰 미팅에서 가치 제안 스토리라인을 검증하고 Feedback Log를 남긴다.
   - 미해결 피드백은 Jira 티켓으로 생성하여 추적한다.

## Deliverables
- 세그먼트별 Value Proposition Canvas, JTBD 요약 카드, 인터뷰/설문 데이터셋
- Messaging Pillar 문서, 예시 카피/비주얼, 증거 자산 목록
- 실험 가설 표, 예상 KPI, 의존 페이즈 맵

## Success Metrics
- 핵심 세그먼트 3개 이상에 대해 Pain/Gain/Jobs 필드가 100% 채워진 Canvas 완성
- 경영진 리뷰에서 Rework 요청이 10% 이하이며, 승인 후 48시간 내 배포 가능 상태
- 정의된 가설 최소 2건이 실험 백로그에 등록되고, 필요한 데이터 수집 계획이 첨부됨

## Timeline & Coordination
- D1: 기존 리서치 정리 및 추가 인터뷰 스크립트 확정
- D2~D3: 인터뷰/설문 수행, Value Proposition Canvas 작성
- D4: Messaging Pillar, 실험 가설 정리, 리뷰 자료 패키징
- D5: 리뷰 미팅, 피드백 반영, 산출물 버전 확정 후 phase-016에 공유

## Notes
- 모든 자료는 `docs/records/phase-015/`에 업로드하고, 민감 데이터는 접근 권한을 제한한다.
- 가치 제안 문구는 이후 번역·현지화 작업을 고려해 용어사전(Glossary)와 동기화한다.
- 수집한 설문 원천 데이터는 익명화 후 저장하고, 재사용 시 개인정보 규정을 준수한다.

## Cost & Translation Guardrails
- 모든 작업은 `docs/strategy/translation-guardrails.md`에 명시된 **저비용·로컬 AI 우선** 원칙을 따른다.
- 상용 번역 API는 기본적으로 비활성화하며, 필요 시 이번 페이즈 산출물에 비용/위험을 명시하고 책임 승인 후 사용한다.
- 16GB VRAM RTX 5070 Ti 환경에서 실행 가능한 Hugging Face 모델과 QLoRA/양자화 기법을 우선 고려한다.
- Reddit 등 소스 콘텐츠는 핵심만 추출하여 NewsKoo 스타일로 재작성하고, 저작권 및 약관을 준수했음을 검수 로그에 남긴다.
