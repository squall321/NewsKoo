# AGENTS

이 리포지토리에서 작업할 때는 다음 원칙을 지켜 주세요.

1. **문서 우선**: 새 기능이나 설정을 추가할 때는 `README.md` 또는 각 하위 디렉터리의 README를 갱신하여 맥락을 남깁니다.
2. **양쪽 스택 정렬**: 프론트엔드와 백엔드 모두에 영향을 주는 변경이라면 두 영역의 의존성 파일과 Lint/Format 설정을 같이 검토합니다.
3. **테스트/검증 명시**: 새로운 명령이나 스크립트를 추가하면 실행 예시와 기대 출력을 `docs/` 또는 관련 README에 명시합니다.
4. **형식 도구 준수**: Python은 `black`/`ruff`, 프론트엔드는 `eslint`/`prettier` 구성을 기본으로 사용합니다. 설정을 수정한 경우 사유를 문서화합니다.

위 지침은 리포지토리 전체에 적용됩니다.
